# 📘 User Story - US03

Como professor, desejo associar um tópico a cada aula do semestre para organizar o conteúdo que será ministrado em cada dia.
<br>

## 📌 Explicação

Essa funcionalidade permite que o professor organize o conteúdo da disciplina ao longo das aulas do semestre, associando tópicos aos dias letivos.<br>
Diferente de um processo manual, a distribuição dos tópicos nas aulas deve ser realizada automaticamente, com base na quantidade de aulas disponíveis e na quantidade necessária para cada tópico.<br>
O objetivo é facilitar o planejamento, garantindo consistência entre o conteúdo e a carga horária da disciplina.<br>
<br>

## 📌 Informações exibidas

Para cada aula do semestre, devem ser apresentados:<br>

- Data da aula (ex: 02/03/2026)<br>
- Dia da semana (ex: Segunda)<br>
- Horário da aula (ex: 18:45 - 19:35)<br>
- Evento (ex: Semana 1 sprint, Semana 2 sprint, Planning, feriado, evento acadêmico, etc)<br>
- Tópico associado à aula<br>
<br>

## 📌 Regras de Negócio
- A associação entre tópicos e aulas deve ser realizada automaticamente<br>
- A distribuição deve respeitar a quantidade mínima de aulas definida para cada tópico<br>
- Um tópico pode ocupar uma ou mais aulas consecutivas<br>
- A distribuição deve seguir a ordem dos tópicos cadastrados<br>
- Dias marcados como feriado ou evento não devem receber conteúdo<br>
- Dias marcados como Semana 3 sprint e planning não podem ter avaliações<br>
- A distribuição deve considerar a quantidade de aulas disponíveis por dia<br>
<br>

## 📌 Critérios de Aceitação
- Ao iniciar o planejamento, os tópicos devem ser distribuídos automaticamente nas aulas do semestre<br>
- Cada aula deve apresentar um tópico associado (quando aplicável)<br>
- A quantidade de aulas atribuídas a cada tópico deve respeitar o mínimo definido<br>
- Nenhum tópico deve ser associado a datas não letivas<br>
- A distribuição deve ocorrer de forma contínua e ordenada<br>
<br>
